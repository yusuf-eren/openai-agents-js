    Tracing

    Tracing helps you monitor, debug, and analyze your agents' behavior. It provides detailed insights into how agents make decisions, use tools, and handle handoffs.

    Basic Tracing

    import { Agent, Runner, Tracer } from 'openai-agents';

// Create a tracer
const tracer = new Tracer({
  name: "my-tracer",
  level: "debug"
});

// Create an agent with tracing enabled
const agent = new Agent({
  name: "TracedAgent",
  instructions: "You are a helpful assistant",
  tracer: tracer
});

// Run the agent with tracing
const result = await Runner.run(agent, "What is the weather?", {
  trace: true
});

    Trace Levels

    • debug: Detailed information for debugging
    • info: General operational information
    • warn: Warning messages
    • error: Error messages

    Trace Types

    1. Agent Traces

    Track agent behavior and decisions:

    const tracer = new Tracer({
  name: "agent-tracer",
  level: "debug",
  onAgentStart: (agent) => {
    console.log(`Agent ${agent.name} started`);
  },
  onAgentEnd: (agent, result) => {
    console.log(`Agent ${agent.name} completed`);
  }
});

    2. Tool Traces

    Monitor tool usage and performance:

    const tracer = new Tracer({
  name: "tool-tracer",
  level: "debug",
  onToolStart: (tool, args) => {
    console.log(`Tool ${tool.name} called with args:`, args);
  },
  onToolEnd: (tool, result) => {
    console.log(`Tool ${tool.name} completed with result:`, result);
  }
});

    3. Handoff Traces

    Track agent handoffs:

    const tracer = new Tracer({
  name: "handoff-tracer",
  level: "debug",
  onHandoffStart: (fromAgent, toAgent) => {
    console.log(`Handoff from ${fromAgent.name} to ${toAgent.name}`);
  },
  onHandoffEnd: (fromAgent, toAgent, result) => {
    console.log(`Handoff completed from ${fromAgent.name} to ${toAgent.name}`);
  }
});

    Trace Storage

    1. Console Storage

    Default storage that logs to console:

    const tracer = new Tracer({
  name: "console-tracer",
  storage: "console"
});

    2. File Storage

    Store traces in files:

    const tracer = new Tracer({
  name: "file-tracer",
  storage: {
    type: "file",
    path: "./traces"
  }
});

    3. Custom Storage

    Implement custom storage:

    const tracer = new Tracer({
  name: "custom-tracer",
  storage: {
    type: "custom",
    save: async (trace) => {
      // Custom storage implementation
    }
  }
});

    Advanced Features

    Trace Filtering

    Filter which traces to record:

    const tracer = new Tracer({
  name: "filtered-tracer",
  filters: {
    agents: ["MathAgent", "CodeAgent"],
    tools: ["calculator", "code_analyzer"],
    levels: ["error", "warn"]
  }
});

    Trace Analysis

    Analyze trace data:

    const analysis = await tracer.analyze({
  timeRange: {
    start: new Date("2024-01-01"),
    end: new Date()
  },
  metrics: ["duration", "success_rate", "error_rate"]
});

    Trace Visualization

    Visualize trace data:

    const visualization = await tracer.visualize({
  type: "timeline",
  data: traces
});

    Best Practices

    1. Appropriate Level: Use the right trace level for your needs
    2. Storage Management: Regularly clean up old traces
    3. Performance Impact: Be mindful of tracing overhead
    4. Security: Don't log sensitive information
    5. Analysis: Regularly analyze traces for insights
