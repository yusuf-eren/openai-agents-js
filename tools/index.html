    Tools

    Tools let agents take actions: things like fetching data, running code, calling external APIs, and even using a computer. There are three classes of tools in the Agent SDK:

    • Hosted tools: these run on LLM servers alongside the AI models. OpenAI offers retrieval, web search and computer use as hosted tools.
    • Function calling: these allow you to use any JavaScript/TypeScript function as a tool.
    • Agents as tools: this allows you to use an agent as a tool, allowing Agents to call other agents without handing off to them.

    Hosted tools

    OpenAI offers a few built-in tools when using the OpenAIResponsesModel:

    • The WebSearchTool lets an agent search the web.
    • The FileSearchTool allows retrieving information from your OpenAI Vector Stores.
    • The ComputerTool allows automating computer use tasks.
    import { Agent, FileSearchTool, Runner, WebSearchTool } from 'openai-agents-js';

const agent = new Agent({
  name: "Assistant",
  tools: [
    new WebSearchTool(),
    new FileSearchTool({
      maxNumResults: 3,
      vectorStoreIds: ["VECTOR_STORE_ID"],
    }),
  ],
});

async function main() {
  const result = await Runner.run(agent, "Which coffee shop should I go to, taking into account my preferences and the weather today in SF?");
  console.log(result.finalOutput);
}

main();

    Function tools

    You can use any JavaScript/TypeScript function as a tool. The Agents SDK will setup the tool automatically:

    • The name of the tool will be the name of the function (or you can provide a name)
    • Tool description will be taken from the JSDoc comments of the function (or you can provide a description)
    • The schema for the function inputs is automatically created from the function's arguments using Zod
    • Descriptions for each input are taken from the JSDoc comments of the function, unless disabled

    We use TypeScript's type system and Zod for schema creation and validation.

    import { z } from 'zod';
import { Agent, FunctionTool, Runner } from 'openai-agents-js';

// Define the schema for the weather data
const WeatherSchema = z.object({
  city: z.string(),
  temperatureRange: z.string(),
  conditions: z.string(),
});

// Create a function tool
const getWeather = new FunctionTool({
  name: 'get_weather',
  description: 'Get the current weather for a city',
  on_invoke_tool: async ({ context, input }) => {
    const parsedInput = JSON.parse(input);
    return {
      city: parsedInput.city,
      temperatureRange: '14-20C',
      conditions: 'Sunny with wind.',
    };
  },
  params_json_schema: {
    type: 'object',
    properties: {
      city: {
        type: 'string',
        description: 'The city to get weather for',
      },
    },
    required: ['city'],
  },
  strict_json_schema: false,
});

// Create an agent with the tool
const agent = new Agent({
  name: 'Weather Assistant',
  instructions: 'You are a helpful weather assistant.',
  tools: [getWeather],
});

async function main() {
  const result = await Runner.run(agent, "What's the weather in Tokyo?");
  console.log(result.finalOutput);
  // The weather in Tokyo is sunny.
}

main();

    Custom function tools

    Sometimes, you don't want to use a JavaScript/TypeScript function as a tool. You can directly create a FunctionTool if you prefer. You'll need to provide:

    • name
    • description
    • params_json_schema, which is the JSON schema for the arguments
    • on_invoke_tool, which is an async function that receives the context and the arguments as a JSON string, and must return the tool output as a string.
    import { z } from 'zod';
import { FunctionTool } from 'openai-agents-js';

// Define the schema for user data
const UserSchema = z.object({
  username: z.string(),
  age: z.number(),
});

// Create a custom function tool
const processUser = new FunctionTool({
  name: 'process_user',
  description: 'Processes extracted user data',
  params_json_schema: UserSchema,
  on_invoke_tool: async ({ context, input }) => {
    const parsed = UserSchema.parse(JSON.parse(input));
    return `${parsed.username} is ${parsed.age} years old`;
  },
});

    Agents as tools

    In some workflows, you may want a central agent to orchestrate a network of specialized agents, instead of handing off control. You can do this by modeling agents as tools.

    import { Agent, Runner } from 'openai-agents-js';

const spanishAgent = new Agent({
  name: "Spanish agent",
  instructions: "You translate the user's message to Spanish",
});

const frenchAgent = new Agent({
  name: "French agent",
  instructions: "You translate the user's message to French",
});

const orchestratorAgent = new Agent({
  name: "orchestrator_agent",
  instructions: "You are a translation agent. You use the tools given to you to translate. If asked for multiple translations, you call the relevant tools.",
  tools: [
    spanishAgent.asTool({
      toolName: "translate_to_spanish",
      toolDescription: "Translate the user's message to Spanish",
    }),
    frenchAgent.asTool({
      toolName: "translate_to_french",
      toolDescription: "Translate the user's message to French",
    }),
  ],
});

async function main() {
  const result = await Runner.run(orchestratorAgent, "Say 'Hello, how are you?' in Spanish.");
  console.log(result.finalOutput);
}

main();

    Customizing tool-agents

    The agent.asTool function is a convenience method to make it easy to turn an agent into a tool. It doesn't support all configuration though; for example, you can't set maxTurns. For advanced use cases, use Runner.run directly in your tool implementation:

    import { Agent, FunctionTool, Runner } from 'openai-agents-js';

const runMyAgent = new FunctionTool({
  name: 'run_my_agent',
  description: 'A tool that runs the agent with custom configs',
  on_invoke_tool: async ({ context, input }) => {
    const agent = new Agent({
      name: "My agent",
      instructions: "...",
    });

    const result = await Runner.run(
      agent,
      input,
      {
        maxTurns: 5,
        // ... other run config options
      }
    );

    return result.finalOutput;
  },
});

    Handling errors in function tools

    When you create a function tool, you can handle errors in the on_invoke_tool function. The SDK provides several error types:

    • ModelBehaviorError: If the model produced invalid JSON
    • UserError: If your code crashed
    • ToolError: For general tool-related errors

    You can catch and handle these errors in your on_invoke_tool implementation:

    import { FunctionTool, ModelBehaviorError, UserError } from 'openai-agents-js';

const tool = new FunctionTool({
  name: 'my_tool',
  description: 'A tool that handles errors gracefully',
  on_invoke_tool: async ({ context, input }) => {
    try {
      // Your tool implementation
      return "Success!";
    } catch (error) {
      if (error instanceof ModelBehaviorError) {
        return "The model provided invalid input. Please try again.";
      }
      if (error instanceof UserError) {
        return "An error occurred while processing your request.";
      }
      throw error; // Re-throw unexpected errors
    }
  },
});
